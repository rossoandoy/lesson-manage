# Feedback.md
集計・印刷機能の要件
• 間瀬さんがブース表および印刷・集計に関する要件をまとめて説明した
• ブース表の操作要件
    * コマンドメニューは右側に配置（左側は入力しづらい）
    * セル単体入力後に期間指定で繰り返し反映できるトリガーが必要
    * 生徒名を検索した際に該当セルがハイライト表示される機能が必要
    * 表示時限の範囲指定（例：*13* 時から表示）と拡大縮小が必要
    * 週単位表示をデフォルトとし、翌週・翌々週へのナビゲーションが必要
    * 途中解約時など、特定生徒の指定日以降のコマを一括削除できる機能が必要
    * 休校日の全予定を一括削除できるコマンドも必要
• 出欠管理はチェックボックスではなく、コマンドメニューから出席・欠席・振替を選ぶ形が望ましい
• 印刷機能の要件
    * ブース表印刷は日付範囲を指定して印刷できること（週固定でも可）
    * 入力した全コマをリスト化した「データ出力シート」を用意し、生徒名・講師名・期間でフィルタリングして個別授業予定表として印刷できること
• 集計・回数報告の要件
    * 生徒ごとの集計（予定コマ数・出席・欠席・振替）が必須
    * 振替の未消化分が視覚化できること
    * 前年度末までの累計コマ数を別途保持するシートを設け、毎年のブース表データを参照して重くならない設計にすること

期間指定での一括入力
• 間瀬さんは「特定の曜日・時限・ブースのセルを選択し、何月から何月まで繰り返し反映させる」コマンドが必要だと説明した
• 組み合わせ（講師＋生徒セット）ではなく、講師と生徒をそれぞれ別に期間指定で入力できれば十分との考え
• 既に入力済みのコマがある場合はエラーを吐き出すか、上書きしないルールが必要
